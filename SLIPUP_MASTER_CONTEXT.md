# CURSOR MASTER CONTEXT — SLIPUP OBSERVATORY

You are working on **SlipUp Observatory**.

This is not a productivity app, not a journal, and not a social feed.

SlipUp is an **atmospheric instrument built from human moments**.

The system observes patterns in everyday moments without judging, optimizing, or diagnosing.

Your job is to maintain and evolve this system **without breaking its philosophical and architectural integrity**.

---

# Core Identity

SlipUp models human activity as **weather made of moments**.

The user does not post content.
The user contributes a **moment**.

Moments accumulate and create an **atmospheric field**.

From this field emerges:

* an atmospheric reading
* local signals
* structural tendencies

The system observes; it does not correct.

---

# The Four Layers of the Observatory

The vertical structure of the product must remain:

```
Atmosphere
↓
Nearby
↓
Horizon
↓
Strata
```

Each layer represents a deeper temporal scale.

## Atmosphere

The shared atmospheric state over the last 48 hours.

Shows:

* the global degree
* a minimal reading line

Example:

```
Atmosphere
Reading from the last 48 hours.

28°

Quiet.
```

Rules:

* no telemetry
* no counts
* no density
* no signal metrics

The sky must feel calm and stable.

---

## Nearby

The nearby human field.

This layer shows **real moments** from a coarse regional bucket.

Purpose:
to reveal that the atmosphere is made from real human fragments.

Example:

```
Nearby

Local field reading.

Quiet.

Moments nearby

Observed · Focus · Working Alone
01:58 AM · Jerusalem

Observed · Stressed · Going To The Gym
03:41 PM · Jerusalem
```

Rules:

* no density
* no "field scope"
* no technical instrumentation
* no trending
* no infinite scroll

Nearby must feel like **listening to the air around you**.

Interpretation must never appear before human signals.

---

## Horizon

The structural reading of the user's moment field.

This layer shows **short observational lines** such as:

```
Forming.
Leans to observed.
Holds.
Shifting.
```

Rules:

* short
* observational
* non-explanatory
* no analytics language

Horizon reads structure but does not explain the model.

---

## Strata

The slowest layer.

Strata represents **sediment of repeated structure over time**.

Example:

```
Strata

Observed calm appears often under stress.
Avoidable tension repeats in late hours.
```

Rules:

* very few lines
* changes slowly
* no metrics
* no history feed

Strata should feel like geological layers.

---

# Contribute Panel Philosophy

The contribute panel is not a posting interface.

It is a **portal where a moment enters the atmosphere**.

Example structure:

```
A moment
Into the atmosphere

Where a moment enters the reading.

Type
Mood

Rise
Name the moment

Share to the atmosphere
Shared moments enter the atmosphere reading.

No account.
No exact pin.
Sharing off keeps this moment on your device.

Contribute
```

Key principles:

* moments must remain short
* moments must remain everyday fragments
* avoid performative writing
* encourage naming the moment, not explaining it

Examples of moments:

```
Working alone
Running late
Too much coffee
Need fresh air
Going to the gym
```

These examples should appear only as **placeholders**, not fixed UI elements.

Nearby must always show **real moments**, never canned examples.

# Climate Model Philosophy

The atmospheric degree is not a score.

It represents **condensation and dispersion of human signals**.

The model must respect:

* mass inertia
* deterministic computation
* server authority

Climate movement must slow as total mass increases.

Example rule:

```
more total mass → lower marginal impact per moment
```

This ensures the atmosphere feels **heavy and physical** at scale.

---

# API Contract

The server is the single source of truth.

`GET /climate` must return:

```
{
  modelVersion,
  referenceTime,
  windowHours,
  computedDegree,
  condition,
  repetition
}
```

Rules:

* computedDegree is the only truth for the degree
* referenceTime must be generated by the server
* client animation uses displayDegree only
* same input + same referenceTime + same modelVersion → same output

---

# Aggregation Model

At scale the system must not read raw moments.

Climate computation should read **48 hourly buckets** for the 48 hour window.

Buckets may include:

* totals
* type × mood aggregates
* pressure signals
* total mass

Never perform full table scans of raw moments for climate.

---

# Privacy Principles

SlipUp minimizes data by design.

Never collect:

* exact location
* user identity
* persistent tracking

Nearby uses only **coarse regional buckets**.

Rules:

* no raw IP storage
* no identity profiles
* append-only moments
* moderation via hidden=true

# Psychological Principles

The system should create three perceptions:

## Field

The user senses that moments exist beyond them.

Example line:

```
Moments rise across the atmosphere.
```

## Contribution

The user feels their moment slightly affects the field.

Micro feedback:

```
The reading adjusts.
```

## Observation

The system reveals patterns without explanation.

Example:

```
Quiet.
Forming.
Leans to observed.
```

---

# Growth Principle

SlipUp begins to feel alive when the 48h window contains roughly **300–500 moments**.

At that point:

* Nearby consistently shows signals
* Horizon becomes stable
* the climate gains inertia
* the system feels atmospheric rather than empty

The key behavioral metric is **moment density**: moments per active contributor in a short window. When it reaches roughly **3–5 moments per person in 48h**, the product starts to feel alive. Growth should emphasize **weight and calm**, not activity.

See `docs/SLIPUP_METRICS_AND_SIGNALS.md` for the full framework (density, return rate, what not to measure, and leading signals).

---

# Golden Rule

SlipUp must feel like **a place to observe the human climate**, not a tool to optimize behavior.

The system should feel less like an app and more like **standing under a shared sky made of moments**.
